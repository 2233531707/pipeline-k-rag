地下管网知识模型数据库
Windows安装与使用教程

一、安装前准备

1. 系统要求
   - Windows 10 或 Windows 11，64 位系统。
   - 建议至少 16 GB 内存，数据量较大时建议 32 GB 或更多。
   - 首次安装建议预留至少 30 GB 磁盘空间，长期使用请预留 60 GB 以上。
   - 首次启动需要联网下载 Docker 镜像和依赖。

2. 安装 Docker Desktop
   - 下载地址：https://www.docker.com/products/docker-desktop/
   - 安装时使用默认选项，并启用 WSL 2。
   - 安装完成后重启电脑。
   - 打开 Docker Desktop，等待 Docker Engine 正常运行。

二、安装本软件

1. 双击“地下管网知识模型数据库.exe”。
2. 按安装向导点击“下一步”和“安装”。
3. 建议保留默认安装位置：
   %LOCALAPPDATA%\地下管网知识模型数据库
4. 安装完成后勾选“启动地下管网知识模型数据库”。
5. 如果 Windows 提示“未知发布者”，原因是当前安装包未购买代码签名证书。
   请确认安装包来源可信后，点击“更多信息”再点击“仍要运行”。

三、第一次启动

1. 保持 Docker Desktop 处于运行状态。
2. 打开桌面的“地下管网知识模型数据库”。
3. 在启动器中按顺序点击：
   ① 环境检查
   ② 初始化配置
   ③ 启动
4. 第一次启动需要下载和构建镜像，通常需要 10 至 40 分钟。
   网络较慢时可能更久，请不要反复点击“启动”。
5. 日志中出现“API 与 Web 健康检查通过”后，点击“打开 Web”。
6. 浏览器也可以手动访问：
   http://localhost:5173
7. 首次进入页面时，按页面提示创建超级管理员账号和密码。

四、配置大模型

推荐在 Web 管理界面的“模型配置”中添加模型服务。

也可以手动配置：
1. 点击启动器中的“停止”。
2. 用记事本打开：
   %LOCALAPPDATA%\地下管网知识模型数据库\app\.env
3. 找到 SILICONFLOW_API_KEY=，在等号后填写自己的密钥。
4. 保存文件，回到启动器点击“启动”。

不要把自己的 API Key、密码或 .env 文件发送给其他人。

五、日常使用

启动：
1. 打开 Docker Desktop。
2. 打开“地下管网知识模型数据库”。
3. 点击“启动”。
4. 健康检查通过后点击“打开 Web”。

停止：
1. 在启动器中点击“停止”。
2. 停止不会删除知识库、图谱、账号或空间数据。

查看问题：
1. 点击“服务状态”确认各服务是否运行。
2. 点击“查看日志”，复制最后的错误信息用于排查。
3. Docker Desktop 未启动时，先启动 Docker Desktop，再重试。

六、导入知识库迁移包

1. 登录 Web 管理界面。
2. 进入“知识模型数据库”。
3. 选择“迁移包导入”。
4. 选择 .yuxikb.zip 文件并等待后台任务完成。
5. 大型迁移包导入时间较长，请不要关闭 Docker Desktop。

七、导入空间数据

1. 进入目标知识库的空间数据页面。
2. 上传 GeoJSON、SHP ZIP 或 GPKG 文件。
3. 等待导入任务完成后，在空间图层页面预览。
4. 地图支持鼠标拖拽、滚轮缩放和点击要素查看属性。

八、数据备份

重要数据位于：
%LOCALAPPDATA%\地下管网知识模型数据库\app\docker\volumes

PostgreSQL/PostGIS 还使用 Docker 命名卷 postgres_data。

建议：
1. 定期使用系统的知识库迁移包导出功能备份知识库。
2. 完整备份前先在启动器中点击“停止”。
3. 不要执行 docker compose down -v。
4. 不要手动删除 docker\volumes 目录或 postgres_data 卷。

九、升级和卸载

升级：
1. 先停止服务。
2. 备份知识库迁移包和 docker\volumes。
3. 运行新版安装包并安装到原目录。
4. 启动器会保留已有 .env，并补充新增配置项。

卸载：
1. 先在启动器中点击“停止”。
2. 在 Windows“设置 → 应用”中卸载本软件。
3. 卸载不会主动执行 docker compose down -v。
4. 确认不再需要数据后，再手动清理残留目录和 Docker 卷。

十、常见故障

1. 提示“未找到 docker 命令”
   - 安装 Docker Desktop。
   - 安装后重启电脑。

2. 提示“Docker daemon 不可用”
   - 打开 Docker Desktop。
   - 等待 Docker Engine 完全启动后重试。

3. 页面打不开
   - 点击“服务状态”和“查看日志”。
   - 确认 5173、5050、5432、6379 等端口没有被其他程序占用。

4. 第一次启动很慢
   - 这是下载镜像和构建依赖造成的正常现象。
   - 保持网络和 Docker Desktop 正常，不要重复启动。

5. 知识图谱或知识库工具不可用
   - 确认 Milvus、Neo4j、API 和 Worker 均处于运行状态。
   - 在智能体编辑页面启用对应知识库和工具。

6. 迁移包上传失败
   - 确认文件扩展名为 .yuxikb.zip。
   - 检查磁盘剩余空间。
   - 查看启动器日志中的 API 错误信息。
