# 2026-06-30 桌面前端客户端第三轮 Grill

## 背景

第一轮已确认 Windows 交付方向从本地单机版切换为 Electron 桌面前端客户端，客户端连接独立后端服务器，不托管后端、数据库、对象存储、向量库、图数据库、Worker 或智能体沙盒。

第二轮已完成主智能体对话闭环桌面化，覆盖附件、预览/下载、知识库来源、会话地图入口、切换服务器状态清理，并完成 Electron portable 重打包、GUI 验收补录和空间地图后端阻塞修复。

第三轮目标从“继续补业务功能”调整为：围绕前后端分离交付，补齐免安装桌面前端客户端、独立后端服务器配套部署、桌面诊断底座，并建立 Web 前端能力桌面化全量等价的后续任务矩阵。

## 最终目标

前后端分离交付：

- 前端以免安装解压式 Electron 桌面前端客户端交付。
- 后端单独部署在服务器上，承载 API、Worker、数据库、对象存储、向量库、图数据库和沙盒运行能力。
- 桌面前端客户端应逐步达到 Web 前端能力桌面化全量等价：Docker 中 Web 前端入口能完成的产品能力，Electron 客户端也应能完成；差异只应存在于运行外壳、后端地址配置、桌面端认证存储和免安装交付方式。

## 已确认边界

- 桌面前端客户端不是本地单机版，不是后端启动器，不管理 Docker、Compose、数据库或沙盒。
- 第三轮不做安装器、自动更新、静默安装或企业分发体系。
- 第三轮不承诺一次性补齐 Web 端全部功能桌面化。
- 第三轮不做客户端内置后端、本地代理、本地静态文件服务或客户端托管 API 转发层。
- 第三轮不做 Kubernetes、云市场镜像或自动化服务器安装向导。
- 第三轮不诊断模型 API Key、知识库数据质量、智能体工具执行失败或空间数据导入失败；这些属于后端业务诊断。

## 第三轮 P0 范围

### 1. 免安装桌面前端交付包

交付物从单文件 portable exe 调整为“zip 解压目录 + exe”的绿色版交付。

建议产物：

```text
地下管网知识模型数据库-桌面客户端-vX.Y.Z/
  地下管网知识模型数据库.exe
  resources/
  locales/
  version.json
  README.txt
  LICENSES.chromium.html
```

验收：

- 产出 `地下管网知识模型数据库-桌面客户端-vX.Y.Z.zip`。
- 解压后双击 `地下管网知识模型数据库.exe` 可启动。
- 移动整个解压目录后仍可启动。
- 解压目录不包含 backend、docker、server、database、Compose、数据库、Docker 镜像或服务端配置。
- `version.json` 记录客户端版本、构建时间、桌面协议版本和 git commit。
- `version.json` 缺失时客户端仍可启动，但诊断信息提示版本元数据缺失。

### 2. 连接与诊断底座

第三轮诊断范围锁定为客户端能直接判断的连接与运行前置问题：

- 后端地址不可达。
- `/api/system/health` 不通过。
- CORS 或 `kb-desktop://app` 未放行。
- HTTPS / 证书问题，只提示和记录，不默认绕过证书校验。
- 认证态异常，例如 token 过期、切换服务器后旧 token 残留、当前用户信息拉取失败。
- 前后端协议版本不兼容。

诊断入口：

- 连接配置页：连接失败时显示“导出诊断信息”，连接成功时也允许导出用于交付验收留档。
- 登录后用户菜单或设置入口：显示客户端版本、后端地址、后端版本、协议版本、最近连接检查结果，并允许导出诊断。

诊断导出：

- 第三轮先导出白名单 JSON 文件，不做 zip 诊断包。
- 文件名建议：`yuxi-desktop-diagnostics-YYYYMMDD-HHmmss.json`。
- 不自动上传到任何服务器。
- 推荐使用 Electron 保存对话框导出 JSON。
- Web 端不显示该入口，或只提供复制诊断信息。

诊断白名单允许包含：

- 客户端版本、Electron/Chrome/Node 版本、Windows 平台信息。
- 客户端桌面协议版本。
- 后端地址 origin，不包含敏感 query。
- `/api/system/health` 检查状态、HTTP 状态码、错误类型。
- 后端版本、后端桌面协议版本。
- 最近诊断事件：连接检查、登录态清理、版本不兼容、CORS/证书错误类型。
- 时间戳和时区。
- 是否桌面模式、是否配置后端地址、是否存在登录态；登录态只记录 boolean。

诊断导出禁止包含：

- auth token、cookie、Authorization header。
- 用户名、密码、邮箱、手机号。
- 聊天消息、附件内容、知识库文件内容。
- API Key、模型 provider 配置、数据库连接串。
- 完整后端 URL 的敏感 query 参数。
- 原始响应体，除非是明确白名单字段，例如 `status`、`version`、`desktop_client_protocol_version`。

### 3. 桌面协议版本契约

新增最小桌面协议版本契约：

- 客户端声明 `desktop_client_protocol_version`。
- 后端通过系统信息接口声明 `desktop_client_protocol_version`。
- 协议版本一致时允许继续登录。
- 后端缺少协议版本时提示“后端版本过旧或不支持桌面客户端诊断”。
- 协议版本不一致时提示需要升级客户端或后端，并阻止继续登录。
- 诊断导出记录客户端版本、后端版本、客户端协议版本和后端协议版本。

第三轮不做复杂 feature flags。

### 4. 独立后端服务器部署配套方案

第三轮把 Docker Compose 独立后端服务器部署配套方案纳入 P0，作为 Electron 前端交付包的服务端配套文档与验收清单。

应补齐：

- `docker-compose.prod.yml` 作为服务器部署事实来源。
- `.env.prod` 必填项清单。
- `CORS_ALLOW_ORIGINS=kb-desktop://app` 配置说明。
- 推荐 HTTPS 反向代理说明。
- `/api/system/health`、系统信息接口、桌面协议版本检查说明。
- 后端启动、停止、升级、备份、日志查看最小命令。
- 前端 zip/exe 如何填写服务器地址。
- 端到端验收清单：服务器启动成功 -> 桌面客户端检查连接 -> 登录或初始化 -> 主智能体对话。

不纳入：

- 客户端启动/停止服务器。
- 客户端内置后端。
- 桌面端管理 Docker。
- Kubernetes 部署。
- 自动化服务器安装向导。
- 数据库迁移/备份的完整企业级方案。

### 5. Web 前端能力桌面化全量等价差距矩阵

第三轮必须建立差距矩阵，但不要求一次性实现 P1-P3 全部功能。矩阵采用“页面功能入口为主、技术底座为辅”的双层结构。

字段：

| Web 功能入口 | Electron 当前状态 | 差距 | 依赖底座 | 后续任务 | 优先级 |
|---|---|---|---|---|---|

优先级规则：

- P0：阻断前后端分离交付的能力。
- P1：普通用户核心业务闭环。
- P2：管理员配置与内容生产。
- P3：后台治理和增强体验。

第三轮实现范围限制为 P0 + 差距矩阵；P1-P3 记录为后续任务。

## P0 验收标准

- 能构建绿色版 zip，解压后双击 exe 启动。
- zip 包不包含任何服务端运行内容。
- 客户端能配置独立后端服务器地址。
- 客户端能检查 `/api/system/health`。
- 客户端能读取后端版本和桌面协议版本。
- 桌面协议版本不兼容时阻止继续登录并给出明确提示。
- 连接失败、CORS、证书、健康检查失败、认证态异常均有可诊断提示。
- 连接配置页和登录后入口均可导出严格白名单诊断 JSON。
- 独立后端服务器 Docker Compose 部署文档能指导交付人员启动服务器并让 Electron 客户端连接。
- 文档中包含端到端验收清单。
- 生成 Web 前端能力桌面化全量等价差距矩阵，并记录 P1-P3 后续任务。
