# 知识库分组需求对齐

## 状态

Grill 已完成。本文记录“知识库分组”功能的已确认术语、范围和验收边界，后续实现应按本文拆分任务并优先采用 TDD。

## 目标

在知识库页面增加树形组织层，让管理员可以按“知识库分组 / 子分组 / 知识库”的形式管理知识库。知识库分组仅用于页面归类和创建归属选择，不参与权限、检索或智能体运行语义。

## 已确认术语

- 使用“知识库分组”作为标准术语。
- 不使用“一级文件夹”“目录”“知识库文件夹”描述该能力。
- 知识库仍是权限边界和治理单位。

## 已确认范围

- 知识库分组是全局组织结构，所有管理员看到一致的树形分组结构。
- 默认存在真实落表的“默认分组”，历史知识库和未指定分组的新知识库归入该分组。
- “默认分组”不能重命名、不能删除。
- 管理员可以在任意分组下新建子分组、重命名用户创建的分组、删除空分组。
- 分组名称必填，保存前去除首尾空格，全局大小写不敏感唯一。
- 创建知识库时可以选择分组，默认选择“默认分组”。
- 从迁移包导入知识库时可以选择分组，默认选择“默认分组”。
- 迁移包携带分组名称，导入时复用或创建对应分组；导出不携带分组 ID。
- 已有知识库可以移动到另一个分组。
- 知识库页面按树形分组展示，表现形式类似文件夹，可展开/收起。
- 分组标题显示分组名和当前过滤条件下的知识库总数量（包含子分组）。
- 空分组也显示。
- 默认分组固定在最前面，其他分组按稳定规则排序。
- 每个分组标题提供“新建子分组”和“新建知识库”快捷入口，打开弹窗并默认选中当前分组。
- 页面顶部“新建知识库”仍保留，默认选中“默认分组”。
- 搜索和知识库类型筛选继续作用于所有知识库。
- 不做拖拽移动或拖拽排序。
- 智能体配置、知识库工具、`list_kbs` 和检索接口不感知知识库分组。

## 不纳入本期

- 分组级权限、继承权限或授权。
- 按分组检索或把分组暴露为智能体工具参数。
- 个人分组、个人排序或用户偏好。
- 删除非空分组并自动迁移知识库，或删除仍有子分组的父分组。
- 拖拽移动知识库、拖拽排序分组或知识库。
- 展开状态持久化或个性化视图偏好。

## 验收标准

- 旧实例升级后存在“默认分组”，所有历史知识库都显示在默认分组下。
- 管理员可以创建一个新分组，重复名称会被拒绝。
- 管理员可以在某个分组下创建子分组，刷新后仍显示在父分组下。
- 管理员可以重命名用户创建的空或非空分组，重命名为重复名称会被拒绝。
- 管理员可以删除空的用户创建分组，不能删除非空分组，也不能删除仍有子分组的父分组。
- 管理员不能重命名或删除“默认分组”。
- 新建知识库选择某个分组后，创建成功的知识库出现在该分组下。
- 从迁移包导入知识库选择某个分组后，导入成功的知识库出现在该分组下。
- 已有知识库移动分组后，列表刷新即可看到它出现在目标分组下。
- 搜索和类型筛选后，页面仍按树形分组显示匹配结果，空分组仍保留可见。
- 分组节点支持展开/收起，子分组展示行为与文件夹一致。
- 智能体可见知识库、知识库工具调用和检索行为与分组前保持一致。
