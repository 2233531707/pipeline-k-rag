# 首版桌面前端客户端采用 Electron、自定义协议和独立后端连接

首版 Windows 桌面前端客户端采用 Electron 实现，不采用 Tauri。客户端内置前端构建产物，通过 `kb-desktop://app` 自定义协议加载页面，并连接独立部署的后端服务器。客户端只负责后端地址配置、桌面端安全存储、登录入口和主智能体对话入口，不托管本地代理、本地静态文件服务或任何业务后端能力。

这个决策的主要取舍如下：

- 相比 Tauri，Electron 能更直接复用现有 Vue 3 + Vite 前端，并更快落地首版 `portable exe`。
- 相比 `file://` 或本地 HTTP 静态服务，`kb-desktop://app` 能为桌面端提供稳定且可声明的来源，便于后端通过 `CORS_ALLOW_ORIGINS` 显式放行。
- 相比继续沿用普通前端 `localStorage`，桌面端 token 使用 Electron 原生安全存储，降低凭据暴露风险。
- 相比引入本地代理维持 `/api` 相对路径语义，首版明确让桌面模式走独立后端地址，避免客户端重新承载服务层语义。

首版暂不纳入 OIDC 登录、附件上传、工作区和知识库等全量桌面化能力；这些能力后续如需支持，应继续以“桌面前端客户端连接独立后端服务器”为前提扩展。
